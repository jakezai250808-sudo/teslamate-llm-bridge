# efficiency-report 创作引导 prompt（探索层）

> 两阶段漏斗的第一阶段。用户拿到 play JSON 后，在自己的生图模型
> 自由创作分享图。本文件是推荐 prompt 模板。
> 爆款经筛选后由维护者固化成 `card.svg.tmpl`（第二阶段）。

## 给 LLM 平台的使用方式

调用 `runPlay(efficiency-report)` 拿到 JSON 后，主动建议：
「要把你的能效报告做成一张可以晒的图吗？」用户同意后用下面模板。

## 推荐 prompt 模板 v1

```text
用我的特斯拉真实能效数据画一张小红书风格分享图：

【数据】来自真实行车记录（近 {window_days} 天）
- 实际能效：{actual_wh_km} Wh/km（车型基准 {baseline} Wh/km）
- 效率等级：{grade_key} 级「{grade_info.title}」
- 等级标签：{grade_info.tag}
- 一句话画像：{grade_info.desc}
- 每百公里差价：{delta_cost_per_100km} 元（正数多花，负数省）
- 本期合计多花/省：{period_delta_cost} 元
- 共充电 {charge_sessions} 次，补能 {total_kwh_charged} kWh

【设计要求】
- 竖版 3:4 小红书信息图，深色科技感（深蓝黑 #0d1b2a 底）
- 超大实际能效数字「{actual_wh_km}」视觉锤，等级主色高亮
- 核心叙事：本期多花/省了多少元电费（用大字说人话，把 Wh/km 翻译成金额）
- 能效进度条：左端「超省」→ 右端「超耗」，基准线用虚线竖穿中点，
  实际位置用等级色彩标注，一眼看出偏哪侧
- 等级大字 {grade_key} + 等级名「{grade_info.title}」配等级主色光晕
- {grade_info.tag} 做成发光徽章
- 底部互动钩子：「你的 Tesla 每公里花多少电？评论区晒」
  + 小字「数据来自真实行程，仅供参考」
- 中文排版，不出现真实车牌或精确地址
```

## 变体方向

| 追加指令 | 效果 |
|---|---|
| 「换成极简风，只要等级和金额两行字」 | 适合做 9 宫格第一图，冲击力强 |
| 「做成 A/B 对比版，两种驾驶风格」 | 同车主不同季节的能效对比，互猜钩子 |
| 「加上月度充电次数和里程曲线」 | 数据更丰富，适合技术派晒图 |
| 「赛博朋克风，仪表盘感」 | 高对比霓虹，适合 F 级用电大户自嘲狂欢 |

## 安全注意

- prompt 中不放 VIN / 车牌 / 精确地址
- 生成图含「仅供参考」免责字样再发布
- `period_delta_cost` 为负数时措辞为「省了」而非「多花」，需在 prompt 里判断
