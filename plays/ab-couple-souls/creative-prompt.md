# ab-couple-souls 创作引导 prompt（探索层）

> 两阶段漏斗的第一阶段：用户拿到 play JSON 结果后，用自己的生图模型
> （GPT Image / 豆包生图 / 即梦）自由创作分享图。本文件是推荐 prompt 模板——
> 好 prompt 决定好图的下限。爆款设计经筛选后由维护者固化成 `card.svg.tmpl`
> （第二阶段，见 `docs/play-creative-pipeline.md`）。
>
> 本玩法互动传播力最高（运营手册 #1 金矿）：「猜猜哪个是你哪个是TA」天然
> 召唤 @对方 评论，一张卡顶两张发。双人对比图也比单人图有更高记忆锚点。

## 给 LLM 平台的使用方式

LLM（ChatGPT / 豆包 bot）调用 `runPlay(ab-couple-souls)` 拿到 JSON 后，
应主动建议用户：「要把两个灵魂做成一张对比图吗？发给TA猜猜哪个是自己？」
用户同意后，把下面模板的 `{占位符}` 用 JSON 里的真实值填充，调用生图能力。

## 推荐 prompt 模板 v1（手账贴纸风，实测双卡验证）

```text
用我的真实行车数据画一张小红书风格的「双灵魂对比」分享图（生成图片）：

【数据】同一辆特斯拉的两个驾驶风格灵魂
- 分割线：最高车速中位数 {speed_median} km/h
- 灵魂 A「{a_persona.name}」 {a_persona.tag}
  - 出行 {a_count} 次 · 平均单程 {a_avg_km} km
  - 激进度 {a_aggro}/100 · 夜驾占比 {a_night_pct}%
- 灵魂 B「{b_persona.name}」 {b_persona.tag}
  - 出行 {b_count} 次 · 平均单程 {b_avg_km} km
  - 激进度 {b_aggro}/100 · 夜驾占比 {b_night_pct}%
- 共 {total_count} 次出行记录

【设计要求】
- 竖版 3:4 小红书爆款信息图
- 主视觉：左右两栏，左栏冷蓝调（A 温柔）右栏暖橙红调（B 激进），中缝超大「VS」字
- 两个灵魂各有人物剪影：A 是慵懒开车姿态，B 是专注握方向盘姿态
- 每栏核心数字：激进度大字 + 标签徽章（贴纸质感）
- 底部超大互动钩子：「猜猜哪个是你哪个是TA · 评论区 @TA 来对号入座」
- 小字免责：「数据来自真实行程 · AI 推理仅供娱乐 · 不含 GPS 定位」
- 手账贴纸风：圆角色块、彩色划线、小标签贴纸感、适度留白
- 中文排版，信息密度高但层次清晰
- 不要出现真实车牌 / 具体地名
```

## 变体方向（用户可追加一句话切换风格）

| 追加指令 | 效果 | 适用场景 |
|---|---|---|
| 「赛博朋克风」 | 霓虹高对比 + 电路纹路背景 | B 灵魂激进度 >= 80 时爆感更强 |
| 「极简风」 | 纯色大字 + 最小信息密度 | 适合做头像 / 双人壁纸 |
| 「动漫卡通风，两个灵魂各设计一个角色」 | 角色 IP 感，适合年轻用户 | 家庭共驾双人出圈 |
| 「日系杂志风，黑白配色 + 红色点缀」 | 高级感 | 车主群体高端定位 |
| 「加上一句点评：两个灵魂哪个更省电」 | 顺带晒能耗数据 | 配合 charging 玩法联动 |

## 运营使用说明

- **发布钩子设计**：正文配「猜猜哪个是你哪个是TA」让被@的那人不得不回复，
  天然 2 倍互动量；评论区讨论「谁是A谁是B」延长热度尾巴。
- **最佳发布时机**：周末家庭外出当天或次日（行程记忆鲜活），或节假日返程后。
- **搭配说明文字模板**：
  「车机数据不会说谎，同一辆车跑出来的两种风格，你们分别是哪个灵魂？」
- **适用用户画像**：夫妻同车 / 父母子女共驾 / 合租室友拼车，
  任何两人共用同一辆 Tesla 的场景都可触发。
- **隐私保护提示**：prompt 不含 VIN / 车牌 / 精确地址；激进度由聚合统计算出，
  不暴露单次行程轨迹；发布图加「AI 仅供娱乐」免责文字。

## 安全注意

- prompt 中不放 VIN / 车牌 / 精确地址
- 激进度是区间统计，不代表违规超速判断
- 夜驾比例来自本地时间聚合，不暴露具体出行轨迹
- 提醒用户：生成图含「娱乐参考」免责字样再发布
