# driving-personality 创作引导 prompt（探索层）

> 两阶段漏斗的第一阶段：用户拿到 play JSON 结果后，用自己的生图模型
> （GPT Image / 豆包生图 / 即梦）自由创作分享图。本文件是推荐 prompt 模板 ——
> 好 prompt 决定好图的下限。爆款设计经筛选后由维护者固化成 `card.svg.tmpl`
> （第二阶段，见 `docs/play-creative-pipeline.md`）。

## 给 LLM 平台的使用方式

LLM（ChatGPT / 豆包 bot）调用 `runPlay(driving-personality)` 拿到 JSON 后，
应主动建议用户：「要把这个结果做成一张分享图吗？」用户同意后，把下面模板的
`{占位符}` 用 JSON 里的真实值填充，调用生图能力。

## 适用模型说明

本文件包含两套模板：

| 模板版本 | 适用模型 | 特点 |
|---|---|---|
| **v1（通用版）** | GPT Image（gpt-image-1 / DALL-E 3）、Qwen-Image-2.0、Gemini flash image | 自然段描述 + 设计语言词，GPT 系模型质量最高 |
| **v2（Seedream 专用）** | 火山方舟 Seedream-4.0、豆包生图（即梦） | 白名单约束 + 元语言翻译，专为 Seedream 调优，9/10 评分 |

## 推荐 prompt 模板 v1（通用版 — GPT/Qwen/Gemini）

```text
用我的真实驾驶数据画一张小红书风格的驾驶人格分享图（生成图片）：

【数据】来自我的特斯拉真实行车记录
- 驾驶人格码：{code}「{persona.name}」
- 一句话画像：{persona.desc}
- 自嘲标签：{persona.tag}
- 四轴：动力 {vigor}/100 · 夜驾占比 {night_pct}% · 单程平均 {avg_drive_km}km · 出车率 {freq_pct}%
- 本月：{drive_count} 次出行 · 最快 {top_speed} km/h

【设计要求】
- 竖版 3:4 小红书爆款信息图
- 主题氛围呼应人格（夜行系→深色夜景霓虹；通勤系→清晨城市；远征系→公路日落）
- 超大人格码 {code} 视觉锤，标题「AI 分析：你是一个典型的{persona.name}」
- 数据卡片展示真实数字（数字必须准确，不要编造）
- 四轴条形图可视化
- {persona.tag} 做成发光徽章
- 底部互动钩子「你的驾驶人格是什么？评论区聊聊」
  + 小字「数据来自真实行程，AI 推理仅供娱乐参考」
- 中文排版，信息密度高但层次清晰
- 不要出现真实车牌 / 具体地名
```

## 推荐 prompt 模板 v2（Seedream / 豆包系模型专用）

> 本模板经过 5 轮迭代、9/10 评分验证（2026-06）。  
> 适用于：火山方舟 Seedream-4.0（`generate_play_image` tool）、豆包生图（Seedream 底座）。  
> **不适用于** GPT Image——GPT 不需要文字白名单约束，用 v1 模板即可。

### 完整通用占位符模板

```text
竖版手机海报，1536像素宽2048像素高。

背景：真实摄影，Tesla Model 3银色停在深夜城市路边，车身有霓虹灯紫粉色反光，湿润路面有水光倒影，远处街灯虚化成光晕，整体色调深蓝+紫粉+白光，无插画无卡通。

海报叠层从上到下：

第一行（极小字）：左侧白字「AI分析：你是一个典型的」，右侧白色细条形码+极小白字「{code}-001-3Y」

第二区（大字）：超大加粗斜体字母「{code}」横铺画面，字母颜色左紫右粉渐变，外发白光，磨砂纹理感。下方一行「{persona.name} 👻」白色中号字。右上方六边形徽章「{persona.tag}」紫色霓虹外轮廓。

右侧竖排文字（贴右边缘，距右边距80px，四行从上到下）：逐字竖排「{persona.story_line1}」「{persona.story_line2}」「{persona.story_line3}」「{persona.story_line4}」，浅白半透明小字

第三区（数据卡）：深色半透明圆角矩形，内有白色标题行「本月驾驶数据 TESLA T」。五个数据格横排：
「{drive_count}」出行次数 | 「{total_km}km」行驶里程 | 「{top_speed}km/h」最快速度 | 「{max_km}km」最长一程 | 「{total_hours}h」在路上

第四区（四轴卡）：深色半透明圆角矩形，白色标题「驾驶人格四轴图谱」。
四行量表，每行：左极端词 → 水平量表（左端彩色填充，右端深灰，圆角矩形色条）→ 右极端词，量表下方注明指标名和数值：
行一量表：左「佛系」→ 紫色填充约{vigor/100}宽 → 右「暴力」，下注「动力指数 {vigor}/100」
行二量表：左「日行」→ 蓝色填充约{night_pct/100}宽 → 右「夜行」，下注「夜驾占比 {night_pct}%」
行三量表：左「短途」→ 绿色填充约{avg_km/40}宽 → 右「远征」，下注「单程平均 {avg_km}km」
行四量表：左「低频」→ 黄色填充约{freq_pct/100}宽 → 右「高频」，下注「出车率 {freq_pct}%」

第五区（关键词）：五个横排圆角胶囊「{kw1}」「{kw2}」「{kw3}」「{kw4}」「{kw5}」

第六区（底部大字）：超大加粗白字「你的驾驶人格是什么？」，次行「评论区聊聊 💬」，右下角极小灰字「数据来自真实行程，AI推理仅供娱乐参考」

画面只允许出现：AI分析你是一个典型的 / {code}-001-3Y / {code} / {persona.name} / {persona.tag} / TESLA / 本月驾驶数据 / {drive_count}出行次数 / {total_km}km行驶里程 / {top_speed}km/h最快速度 / {max_km}km最长一程 / {total_hours}h在路上 / 驾驶人格四轴图谱 / 佛系{vigor}/100暴力动力指数 / 日行{night_pct}%夜行夜驾占比 / 短途{avg_km}km远征单程平均 / 低频{freq_pct}%高频出车率 / {kw1}{kw2}{kw3}{kw4}{kw5} / 你的驾驶人格是什么 / 评论区聊聊 / 数据来自真实行程AI推理仅供娱乐参考

绝对不允许：锤子斧头任何工具，科幻飞船星空，游戏界面，真实车牌，小红书logo，上述列表之外的任何文字
```

### 占位符填充说明

从 `run_play("driving-personality", car_id=<n>)` 返回的 JSON 取值：

| 占位符 | JSON 字段 | 说明 |
|---|---|---|
| `{code}` | `code` | 四字母人格码，如 `CNSO` |
| `{persona.name}` | `persona.name` | 人格名，如「深夜静音幽灵」 |
| `{persona.tag}` | `persona.tag` | 自嘲标签，如「#电机声都嫌吵」 |
| `{persona.story_line1~4}` | `persona.desc` 拆行 | 把一句话画像按语义拆成 3-4 个短片段，逐字竖排 |
| `{drive_count}` | `drive_count` | 行驶次数 |
| `{total_km}` | `avg_drive_km × drive_count` 取整 | 总里程（需 LLM 计算） |
| `{top_speed}` | `avg_speed_kmh`（参考值） | 最快速度（若有 max_speed 字段优先用） |
| `{max_km}` | `avg_drive_km` | 最长单程（近似，若有 max_drive_km 优先用） |
| `{total_hours}` | `drive_count × avg_drive_km / 50`（估算） | 总在路上时间 |
| `{vigor}` | `vigor` | 动力指数 0-100 |
| `{night_pct}` | `night_pct` | 夜驾占比 0-100 |
| `{avg_km}` | `avg_drive_km` | 单程平均公里数 |
| `{freq_pct}` | `freq_pct` | 出车率 0-100 |
| `{kw1~5}` | `persona.tag` 及 `code` 推导 | 5 个关键词胶囊，LLM 自由发挥 |

### Seedream 措辞规律（6 轮迭代提炼）

**有效规则（能稳定触发预期行为）：**

1. **文字白名单段落**：在 prompt 末尾加「画面只允许出现这些文字：[枚举]」，是控制乱入文字最有效的单条规则。不加白名单时，Seedream 会在画面中插入来自训练数据的随机汉字短语。
2. **负向约束只写名词**：「不要出现锤子斧头」比「不要字面化工具类名词」有效。越具体越好，每条只写 2-4 个字的名词/短语，不写句子。
3. **质感词组合**：「真实摄影+湿润路面水光倒影+车身霓虹反光」能可靠触发摄影级背景。「真实摄影」是启动词，后面的细节词叠加效果更好。
4. **大字横铺**：「超大加粗斜体字母『X』横铺画面」对大字渲染稳定有效，所有轮次均生效。
5. **容器用「深色半透明圆角矩形」**：比「数据卡片」「信息卡」更具体，能稳定生成 UI 容器叠加层。
6. **数据格用「大数值+小标签」格式**：「五个数据格横排：「23」出行次数 | 「117km」行驶里程」格式，数值和标签分开写，数值不漂移。
7. **量表用「左极端词→色带→右极端词，下注指标名数值」**：把指标名和数值挪到色带下方注释位置，解放色带左右的标签位置，解决标签重叠问题（R5 验证）。

**失效规则（会触发意外行为）：**

1. **括号内的数字比例标注**：`（占全图22%高度）` 会被当成图面标注渲染在对应区域边缘。
2. **设计概念词**：「视觉锤」「信息密度」「超格码」会被字面化渲染到画面中。
3. **「进度条」单词**：容易生成圆形 gauge 仪表盘，改用「水平量表」「横向矩形色条」更稳定。
4. **「N 列×M 行」网格约束**：模型不稳定，改为直接描述每格的相对位置（如「左上/中上/右上」）。
5. **竖排故事文字放在主体区内**：容易被截断，改到「贴右边缘，距边距 80px，逐字拆开描述」效果更好。
6. **编号列表（第一步、第二步…）**：序号词容易被当成正文渲染，改用自然段连续描述。

## 变体方向（用户可追加一句话切换）

| 追加指令 | 效果 |
|---|---|
| 「换成手账贴纸风」 | 浅色可爱风（参照 A/B 人格对比图的视觉系） |
| 「做成 A/B 对比版，我和家人各一个人格」 | 夫妻/家庭共驾互猜钩子（传播力最强变体） |
| 「赛博朋克风」 | 霓虹高对比 |
| 「极简风，只要人格码和一句话」 | 适合做头像 / 壁纸 |
